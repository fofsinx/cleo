# Welcome to Cleo! üéØ

**Cleo** is a robust distributed task queue system for Node.js, engineered to handle background job processing with enterprise-grade reliability and performance. Inspired by Python's Celery but built from the ground up for Node.js, Cleo provides a developer-friendly solution for handling millions of asynchronous tasks at scale.

## ‚ú® Key Features

### üîÑ Task Processing
- **Distributed Execution**: Scale across multiple workers and machines with zero configuration
- **Priority Queues**: Multi-level priority system for critical task handling
- **Batch Processing**: Efficient handling of task groups with shared context
- **Concurrency Control**: Fine-grained control over worker concurrency and resource usage

### üì¨ Message Handling
- **Redis Integration**: Rock-solid message broker with guaranteed delivery
- **Dead Letter Queues**: Automatic handling of failed tasks
- **Message Persistence**: Never lose a task, even during system failures
- **Real-time Updates**: Live task status and progress tracking

### ‚ö° Performance & Scaling
- **Horizontal Scaling**: Add workers dynamically across your infrastructure
- **Load Balancing**: Automatic work distribution among available workers
- **Resource Management**: Smart CPU and memory utilization
- **Connection Pooling**: Efficient database and external service connections

- **TypeScript First**: Full type safety and excellent IDE support
- **Decorators API**: Clean, intuitive task definitions
- **Middleware System**: Easily extend and customize behavior
- **Plugin Architecture**: Rich ecosystem of community plugins

### üìä Monitoring & Debugging

- **Structured Logging**: Detailed task execution logs

## üöÄ Quick Start

```typescript
import { cleo } from '@cleotasks/core';

// Define a task
@cleo.task()
async function processOrder(orderId: string) {
  logger.info('Processing order', {
    fileName: 'examples/quick-start.ts',
    lineNo: 15,
    functionName: 'processOrder',
    variable: 'orderId',
    value: orderId
  });
  
  // Your task logic here
  const order = await OrderService.get(orderId);
  await PaymentService.process(order);
  await NotificationService.notify(order.userId);
}

// Enqueue a task
await cleo.enqueue('processOrder', '12345', {
  priority: 'high',
  delay: '5m',
  retries: 3
});
```
