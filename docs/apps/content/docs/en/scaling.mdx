## üîÑ Load Balancing Strategies

### 1. Round Robin (Default)

```typescript
cleo.configure({
  loadBalancing: {
    strategy: 'round-robin',
    healthCheck: {
      interval: '10s',
      timeout: '2s'
    }
  }
});
```

### 2. Least Connections

```typescript
cleo.configure({
  loadBalancing: {
    strategy: 'least-connections',
    maxConnectionsPerNode: 1000
  }
});
```

### 3. Consistent Hashing

```typescript
cleo.configure({
  loadBalancing: {
    strategy: 'consistent-hash',
    hashKey: (task) => task.groupId // Tasks with same groupId go to same worker
  }
});
```

## üö® Common Pitfalls

1. **Memory Leaks**

```typescript
// ‚ùå Bad: Memory leak
let cache = [];
@cleo.task()
async function leakyTask() {
  cache.push(await fetchData()); // Memory leak!
}

// ‚úÖ Good: Proper cleanup
@cleo.task()
async function efficientTask() {
  const data = await fetchData();
  await processData(data);
  // Data is garbage collected
}
```

2. **Connection Exhaustion**

```typescript
// ‚ùå Bad: New connection per task
@cleo.task()
async function connectionHungryTask() {
  const client = new Redis(); // Don't do this!
}

// ‚úÖ Good: Connection pool
const pool = createConnectionPool();
@cleo.task()
async function efficientTask() {
  const client = await pool.acquire();
  try {
    await processData(client);
  } finally {
    await pool.release(client);
  }
}
```

## üéØ Best Practices

1. **Gradual Scaling**
   - Start small and scale gradually
   - Monitor impact of changes
   - Use canary deployments

2. **Resource Management**

   - Use connection pooling
   - Monitor memory usage

3. **Error Handling**
   - Implement circuit breakers
   - Use proper retry strategies
   - Monitor error rates

4. **Performance Monitoring**

   - Set up alerts
   - Regular performance reviews
